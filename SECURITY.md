# Security Policy

## Supported Versions

The project is in active development. Security fixes are applied to the latest `main` branch.

## Reporting a Vulnerability

If you discover a security issue, please report it privately and do not open a public issue.

Please include:
- Affected component (`Frontend` or `Backend`)
- Impact and risk level
- Reproduction steps
- Suggested mitigation (if available)

Contact: open a private message to repository maintainers via GitHub.

## Response Expectations

- Initial acknowledgement: within 72 hours
- Triage and validation: as soon as reproducible details are available
- Fix timeline: based on severity and exploitability

## Disclosure Policy

Please allow maintainers time to investigate and patch before public disclosure.
